# Event Booking System Refactor Report

## Phase 1 Findings

- Backend is already a Spring Boot Maven application under `backend/src/main/java/com/eventbooking`.
- Frontend is already a Vite React application under `frontend/src`.
- Backend compile succeeds with `mvn -q -DskipTests compile` when run outside the sandbox JVM thread limit.
- Frontend production build succeeds with `npm.cmd run build`.
- Generated artifacts found:
  - `backend/out/` contains compiled legacy `.class` files.
  - `frontend/dist/` is generated by Vite.
- Legacy/static app found:
  - `web/` contains a separate static HTML/CSS/JS implementation and is not referenced by the active Vite frontend or Spring Boot backend.
- Backend package structure issues found:
  - Persistence objects were under `com.eventbooking.model`; target architecture calls for `entity`.
  - AI providers were under `com.eventbooking.service.ai`; target architecture calls for `ai`.
  - Payment and notification modules were implemented as general services rather than module packages.
  - Some service classes still contain implementation logic directly in `service`; this is functionally valid but not a strict service-interface/service-impl split.
- Frontend structure issues found:
  - `services/api.js` exists, but target structure also expects `api`, `routes`, `hooks`, `utils`, `layouts`, `assets`, and `styles` folders.
  - Layout components are under `components/layout`, while target structure expects a top-level `layouts` folder.
- Import issues found:
  - Wildcard imports exist in backend services and should be removed during cleanup.
- Security, JWT, repository, DTO, and exception packages are present and compile.
- Scheduler functionality exists through `@Scheduled` methods in services/security stores, but there is no dedicated `scheduler` package yet.

## Refactor Strategy

- Preserve existing functionality and endpoint contracts.
- Move packages mechanically where target architecture requires it.
- Update imports and package declarations.
- Keep generated artifacts out of source architecture and document them as cleanup candidates.
- Validate with Maven compile and Vite build after changes.
